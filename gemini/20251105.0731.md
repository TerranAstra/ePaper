
You’re getting “permission denied” to the Docker socket. Fix it on the Pi:

- Quick unblock (now):
```bash
sudo docker compose -f docker-compose.mssql.yml --profile pi up -d
```

- Permanent fix (no sudo needed later):
```bash
# Ensure Docker is running
sudo systemctl enable --now docker

# Add your user to docker group
sudo usermod -aG docker $USER

# Apply group now (or log out/in)
newgrp docker

# Verify
docker info
docker run --rm hello-world
docker compose version
```

- Architecture check for Azure SQL Edge (must be 64‑bit ARM):
```bash
uname -m
# aarch64 => OK
# armv7l => not supported by azure-sql-edge (switch to 64‑bit Pi OS or use a different local cache like SQLite)
```

- Rerun:
```bash
docker compose -f docker-compose.mssql.yml --profile pi up -d
```

Notes:
- The “version is obsolete” warning in compose is harmless; you can ignore or remove the version line.
- If you’re on armv7l, azure-sql-edge won’t work; either reimage to 64‑bit Pi OS or use a lightweight local cache (SQLite) on the Pi and keep SQL Server on Windows as the absorber.
